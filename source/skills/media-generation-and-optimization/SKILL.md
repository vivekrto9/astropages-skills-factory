# Produce Efficient Accessible Media

1. Inspect existing media components, aspect ratios, breakpoints, project-asset storage, alt text, loading behavior, and build tooling. Confirm the requested subject, rights, and brand constraints.
2. Create or transform only the smallest requested media set. Keep generated output in the governed project-assets path; do not embed large binaries, data URLs, or external credentials in source.
3. Provide meaningful alt text for informative images and empty alt text for decoration. Avoid generated text inside images when real HTML can carry it.
4. Use measured responsive sizes, modern formats where supported, stable dimensions, lazy loading below the fold, and an intentional focal crop. Preserve source assets when destructive replacement was not requested.
5. Test missing-media fallbacks, narrow and wide crops, loading, layout shift, and build output. Verify optimization with actual byte and dimension evidence.

Report created asset references, rights or generation limitations, and measured results. Do not claim visual approval, authenticity, or performance improvement without evidence.

# Publish Verifiable Discoverability Signals

1. Inspect routes, content bindings, current metadata helpers, canonical rules, robots controls, sitemap generation, structured data, and tests. Trace every proposed field to public source content.
2. Implement the smallest requested metadata or discoverability change. Reuse existing helpers and keep preview, admin, draft, and private URLs out of public indexes.
3. Generate structured data only for a schema type the page actually satisfies. Never invent ratings, prices, availability, credentials, business facts, FAQs, or astrology claims.
4. Keep canonical and alternate URLs environment-safe and based on generic runtime configuration. Preserve EmDash ownership of editable copy.
5. Test escaping, missing optional fields, canonical consistency, sitemap inclusion/exclusion, and rendered JSON-LD validity. Inspect representative pages rather than trusting source objects alone.

Report validation evidence, indexed route scope, and any search-engine behavior that cannot be guaranteed. Do not claim ranking or answer-engine placement.

# Official Contract Record

Capture only what is needed for implementation review:

- provider and requested outcome;
- official HTTPS documentation URLs and retrieval date;
- API or SDK version and deprecation status;
- authentication method, minimum permissions, and credential names to declare;
- server-side operation sequence, schemas, rate limits, timeouts, and error model;
- officially documented webhook authenticity, replay, retry, and ordering controls when applicable, without assuming which mechanism the provider supports;
- consent, retention, deletion, and regional constraints;
- sandbox/live verification plan;
- unresolved facts and resulting unsupported or blocked state.

Do not copy credentials, personal data, full documentation pages, or guessed values into the record.

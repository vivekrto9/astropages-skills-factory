# Work Within the Site's Design System

Improve the requested surface without replacing the site's identity or editorial model.

1. Inventory existing tokens, fonts, spacing, primitives, layouts, breakpoints, and interaction patterns. Extend the existing design system before adding a new abstraction.
2. Preserve approved copy, EmDash bindings, brand inputs, data semantics, routes, and calls to action unless the request explicitly changes them.
3. Build hierarchy with type, spacing, alignment, contrast, and composition. Add decoration only when it supports meaning and remains within scope.
4. Prefer target-project components and CSS conventions. Do not copy Pandit branding, astrology-specific styling, content, or route behavior into generic or unrelated sites.
5. Check the real page with representative content at narrow and wide viewports. Remove ornamental changes that create churn without improving the requested outcome.

Keep the diff reviewable: every changed visual rule must serve the stated goal or a verified responsive/accessibility requirement.
